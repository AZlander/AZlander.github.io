01 · The Foundations

Lab Submissions

Five labs that built up the toolkit behind Grove — from drawing a screen to storing data on the device.

Lab 1

User Interface

First steps with Jetpack Compose and declarative UI.

ReflectionBefore this lab I had some programming experience, but Jetpack Compose was completely new to me. I was used to thinking in terms of logic and functions, so describing UI declaratively took some getting used to. Seeing how composable functions replace XML layouts was a real eye-opener once it finally clicked.

Lab 2

Interaction & State

Making the UI respond to user input through state.

ReflectionThis lab challenged me more than I expected. Understanding how state drives the UI, and why you cannot just update a variable and expect the screen to change, was a real mental shift. I struggled with remember and mutableStateOf at first, but once I saw the UI react to my changes it started to feel intuitive.

Lab 3

Material Design

Polishing the app with Material 3 components and colour.

ReflectionI enjoyed this lab because I could finally see the app looking polished. Applying Material 3 components like cards, buttons, and colour schemes made a big visual difference. The trickiest part was understanding the colour system and knowing when to use primary, surface, or onBackground.

Lab 4

Navigation

Connecting multiple screens with NavHost and routes.

ReflectionNavigation was where things started to feel like a real app. Setting up NavHost, defining routes, and passing arguments between screens was simple in concept but fiddly in practice. I ran into back stack issues and had to debug why some screens were re-creating themselves, which taught me a lot about how Android manages screen history.

Lab 5

Room Database

Persisting data locally with entities, DAOs and Flow.

ReflectionRoom was the most technically demanding lab for me. DAOs, entities, repositories, and coroutines all landed at once, and it took time to see how they connect. The Flow and StateFlow pattern confused me at first, but once I traced the chain from the database table up to the UI it finally made sense.

02 · The Builds

Projects

Grove grew across two projects — first into a structured multi-screen app, then into a fully connected study companion.

Project 1 · Foundation

Grove

Problem Statement

Many students struggle to organise their study goals and track their learning progress in one place. Grove addresses SDG 4 by giving learners a structured space to create study boards, set goals, and stay consistent.

Lessons Learned

Project 1 was my first time building a multi-screen app from scratch using Compose and Navigation. Wiring up the ViewModel so state was shared correctly across screens took more thought than I expected. I also learned how important it is to plan navigation routes early.

View on GitHub
Project 2 · Connected

Grove

Problem Statement

Project 1 gave Grove its structure but it lacked real persistence and live data. Project 2 turns Grove into a fully connected study companion with local Room databases, live API book fetching, and sensor-aware focus modes.

Lessons Learned

The biggest challenge was integrating live book data. I had to pivot to the Open Library API due to Google limits, which proved my architecture was sound. Getting the camera and accelerometer working taught me how deeply hardware permissions affect features.

03 · Looking Back

Reflection & Learning Journey

The hardest moments were usually the most useful ones, which is debugging navigation back stacks, or chasing down why an API call returned nothing.

Looking back from Lab 1 to Project 2, the growth feels real. I started barely understanding how Compose draws a screen and ended up building an app that talks to a database, a cloud server, an external API, and the phone's own hardware. More than any single feature, this course taught me how the layers of a modern app fit together, and that has changed how I look at every app I use.
